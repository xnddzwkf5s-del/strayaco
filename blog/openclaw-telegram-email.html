Connecting Telegram and Email to OpenClaw

OpenClaw Telegram AgentMail Email Automation

A practical guide to giving your OpenClaw AI agent two communication channels — Telegram for instant messaging and AgentMail for email — based on real-world setup.

Overview

Out of the box, OpenClaw listens on a web dashboard at http://localhost:18789. That’s fine for a local terminal, but if you want your agent to message you proactively (reminders, alerts, daily summaries) or receive input while you’re away from your desk, you need channels.

We set up two:

  • Telegram — fast, push-based, great for conversational back-and-forth
  • AgentMail — email inbox for the agent, with scheduled processing and daily summaries

Each has a different personality. Telegram is synchronous — you DM the bot, it replies. Email is asynchronous — the agent checks its inbox on a schedule and processes messages in batches.

Telegram Setup

Telegram is the most popular OpenClaw channel for good reason: it’s free, push-based, works on any device, and the bot API is mature.

1. Create a Bot with BotFather

Open Telegram and message @BotFather:

/newbot

Follow the prompts. You’ll get a token like 123456789:ABCdefGHIjklMNOpqrsTUVwxyz.

2. Configure OpenClaw

Add to ~/.openclaw/openclaw.json:

{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz",
      "dmPolicy": "pairing",
      "groups": {
        "*": {
          "requireMention": true
        }
      }
    }
  }
}

If you prefer not to put the token in plaintext config, use the env var TELEGRAM_BOT_TOKEN.

3. Pair Your Account

Start the gateway, then DM your bot and run:

openclaw pairing list telegram
openclaw pairing approve telegram <CODE>

Pairing codes expire after 1 hour. Once paired, you can DM the bot from anywhere.

4. Find Your Telegram User ID

For configuring allowlists, get your numeric user ID:

curl "https://api.telegram.org/bot<token>/getUpdates"

Look for from.id in the JSON. Or DM your bot and check openclaw logs --follow.

5. (Optional) Add to a Group

Add the bot to a group. By default it only responds when mentioned (@botname). To let it see all messages:

  • Make the bot a group admin, or
  • Tell BotFather: /setprivacy → Disabled

AgentMail Setup

AgentMail is an API-first email platform designed for AI agents. Unlike Gmail or Outlook, there’s no OAuth flow, no IMAP, no polling overhead — just a REST API and webhooks.

1. Create an Account

Go to console.agentmail.to and sign up. Generate an API key from the dashboard.

2. Install the SDK

pip install agentmail python-dotenv

3. Create an Inbox

from agentmail import AgentMail

client = AgentMail(api_key="am_us_YOUR_API_KEY_HERE")
inbox = client.inboxes.create(
    username="my-assistant",  # Creates my-assistant@agentmail.to
    client_id="unique-id"
)

You can also create inboxes via the OpenClaw skills config:

{
  "skills": {
    "entries": {
      "agentmail": {
        "apiKey": "am_us_YOUR_API_KEY_HERE"
      }
    }
  }
}

4. Configure the Inbox ID

INBOX_ID = "your-assistant@agentmail.to"

This is where all incoming email will land.

5. Sending Email

client.inboxes.messages.send(
    inbox_id="your-assistant@agentmail.to",
    to="you@example.com",
    subject="Daily Summary",
    text="Here's what happened today..."
)

AgentMail supports attachments (base64-encoded), HTML content, and multiple recipients.

6. Checking Email

The OpenClaw agentmail skill ships with scripts:

# Check the last 10 messages
python3 check_inbox.py --inbox "your-assistant@agentmail.to"

# Monitor for new messages (poll every 30s)
python3 check_inbox.py --inbox "your-assistant@agentmail.to" --monitor 30

Practical Patterns

Pattern 1: Scheduled Email Check (Cron)

The most practical pattern for email is a weekly cron job. The agent:

  1. Checks the inbox for new messages
  2. Marks processed emails as read
  3. Trashes old read emails (older than 14 days)
  4. Summarises anything that needs attention
{
  "schedule": {
    "kind": "cron",
    "expr": "0 8 * * 1",
    "tz": "Australia/Melbourne"
  },
  "payload": {
    "kind": "systemEvent",
    "text": "Time for the weekly AgentMail check. Process the inbox."
  }
}

Pattern 2: Deduplication

Since the agent may check email multiple times before processing, track processed email IDs:

PROCESSED_FILE = "processed_emails.json"

def load_processed():
    try:
        with open(PROCESSED_FILE) as f:
            return json.load(f)
    except:
        return []

def save_processed(ids):
    with open(PROCESSED_FILE, 'w') as f:
        json.dump(ids[-1000:], f)  # Keep last 1000

Pattern 3: Proactive Notifications

Combine Telegram + email notifications. For example:

  • Telegram: Quick alerts ("GitHub push failed", "Server rebooting")
  • Email: Longer summaries ("Weekly email processing report", "Backup status")

The agent can send to either channel depending on message type:

Telegram: short, urgent, needs attention now
Email: detailed, archival, can wait

Pattern 4: Webhook Processing (Advanced)

For real-time email processing, set up a webhook endpoint. When email arrives, the webhook triggers the agent immediately instead of waiting for the next cron run.

webhook = client.webhooks.create(
    url="https://your-server.your-domain.com/webhook/agentmail",
    client_id="email-processor"
)

Warning: Email webhooks are a prompt injection vector. Anyone can email your agent. Use allowlist filtering or isolated sessions to sandbox untrusted senders.

Key Lessons Learned

Telegram:

  • The bot token is sensitive — if exposed, anyone can control your bot. Use env vars or a secrets manager.
  • Pairing codes expire after 1 hour — complete the flow promptly.
  • Group bots need requireMention: false or admin status to see all messages.
  • Telegram has no delete/edit of sent messages via the API that users won’t see.

AgentMail:

  • There is no delete method in the SDK (v0.4.15). Use the trash label instead for cleanup.
  • The API key controls everything — guard it like a password.
  • AgentMail inboxes are public — anyone can email your agent address. Use allowlists or webhook transforms to filter.
  • Processing emails via cron is simpler than webhooks for personal use, and avoids prompt injection risks.

General:

  • Always restart the gateway after config changes: openclaw gateway restart
  • Test each channel independently before relying on it
  • Start with one channel, get it solid, then add the second
