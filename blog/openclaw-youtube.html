Watching YouTube Videos Through Your AI Agent

OpenClaw YouTube Transcript AI Agent yt-dlp

How to give your OpenClaw AI agent the ability to fetch and analyse YouTube video transcripts — for summarisation, Q&A, and content extraction.

Overview

One of the most useful capabilities you can add to an AI agent is the ability to watch YouTube videos. Not literally watch — but fetch the transcript (closed captions or auto-generated subtitles) and work with the text content.

This means you can:

  • Ask your agent to summarise a 30-minute video in a few paragraphs
  • Extract specific information without scrubbing through the timeline
  • Search across multiple videos for a topic
  • Get the key takeaways from a conference talk or tutorial

All with a single command.

How It Works

The setup has three layers:

  1. yt-dlp — a powerful command-line tool that downloads transcripts from YouTube
  2. A Python script — wraps yt-dlp, cleans the WebVTT format into readable text
  3. OpenClaw skill — makes it available as a tool the agent can call

When you ask the agent about a YouTube video, it:

  1. Calls get_transcript.py with the video URL
  2. yt-dlp extracts the closed captions or auto-generated subtitles
  3. The Python script strips timestamps, headers, and formatting noise
  4. Returns clean, readable text to the agent
  5. The agent summarises or answers your question

Installation

1. Install yt-dlp

brew install yt-dlp

Or via pip:

pip install yt-dlp

2. Install the Skill from ClawHub

openclaw skill install youtube-watcher

Or install directly:

openclaw skill install https://clawhub.ai/Michaelgathara/youtube-watcher

3. Verify It Works

python3 /path/to/skills/youtube-watcher/scripts/get_transcript.py "https://www.youtube.com/watch?v=VIDEO_ID"

If the video has captions, you'll see clean text output.

Usage Examples

Summarise a Video

You: Summarise this video: https://www.youtube.com/watch?v=dQw4w9WgXcQ
Agent: [fetches transcript, reads, responds with summary]

Find Specific Information

You: What did the speaker say about error handling in https://youtube.com/watch?v=xyz
Agent: [fetches transcript, searches for "error handling", responds with relevant sections]

Compare Multiple Videos

You: Compare the key arguments in video A and video B
Agent: [fetches both transcripts, compares, responds with analysis]

Why Transcripts Instead of Audio/Video?

Processing raw video or audio would require:

  • Downloading potentially hundreds of megabytes
  • Running a speech-to-text model (computationally expensive)
  • Dealing with different audio formats and quality

Transcripts are:

  • Lightweight — a 30-minute video yields about 5-10KB of text
  • Fast — yt-dlp extracts captions in seconds
  • Accurate — official captions or YouTube's auto-generated subtitles are reliable
  • Searchable — text is easy to search, compare, and process

YouTube Skills Landscape

There's essentially one dedicated YouTube skill for OpenClaw: youtube-watcher from ClawHub. It uses yt-dlp to extract transcripts. But there are alternative approaches, each with different tradeoffs.

Option 1: youtube-watcher Skill (Recommended) ✅

Approach: Install the ClawHub skill + yt-dlp. The agent calls a Python script that wraps yt-dlp to fetch and clean transcript text.

ProsCons
No API key requiredOnly works if video has captions
Fast — returns in secondsNo timing context in output
Ready to use, no custom codeNot suited for videos without CC
Works with any YouTube URL formatEnglish-optimised
Lightweight (text only, no download)No thumbnail/channel metadata

Option 2: Agent-Browser (Custom) ⚠️

Approach: Use the agent-browser skill to navigate to YouTube, read the video page, and copy info from the description and comments. Doesn't get the transcript.

ProsCons
Gets page metadata (title, description, comments)Cannot access transcript or captions
No additional tools neededSlower and fragile (page layout changes)
Can scrape playlists or channelsHigh token usage per video

Not useful for summarising the video itself — only for metadata.

Option 3: YouTube Data API (Custom) 🛠️

Approach: Register for a Google API key, enable the YouTube Data API v3, and write a custom skill.

ProsCons
Official, reliable dataRequires API key + Google Cloud project
Captions, metadata, search in one placeFree tier quota (10,000 units/day)
Returns structured JSONMore complex setup
Includes timing info with captionsQuota management overhead

Overkill for transcript summarisation. Better for building a video search engine or analytics dashboard.

Option 4: Direct yt-dlp (No Skill) 🔧

Approach: Skip the skill and invoke yt-dlp directly via command tool. Same underlying technology, different interface.

ProsCons
No skill installation neededNo guardrails or error handling
Full yt-dlp power (audio download, format selection)Must construct commands manually
Multiple subtitle formats (vtt, srt, ttml)Agent may hit options it doesn't understand

Recommendation

Use youtube-watcher as your primary method. It's the simplest, fastest, and most reliable option for the common use case: summarise a video or ask what it said about a topic.

If you need timestamped transcripts for quoting or precise references, extend the skill — modify get_transcript.py to output timestamps alongside text. The underlying yt-dlp already has this data; the current script strips it for cleaner summarisation.

If you need to process videos without captions (e.g. music videos, raw recordings), the only real option is downloading audio and running it through a transcription model like Whisper. That's a different use case entirely — significantly more resource-intensive.

Limitations

  • No captions = no transcript. Some videos have subtitles disabled. The script fails gracefully.
  • Language dependent. Auto-generated captions are in the video's audio language.
  • Live streams. May not have captions available after the stream ends (varies).
  • Members-only videos. YouTube restricts caption access for some content.
  • Timestamp alignment. The raw transcript is flat text — you lose timing context unless you preserve it.

The last point is worth noting. If you need to know when something was said (e.g. for quoting), you may want a modified script that keeps timestamps. For summarisation, the flat text is fine.

Practical Tips

Tip 1: Check for captions first

Before asking your agent to process a long video, check if captions exist:

yt-dlp --list-subs "https://youtube.com/watch?v=VIDEO_ID"

Tip 2: Adjust for very long videos

Hour-long conference talks generate substantial transcripts. The agent's context window may limit how much it can process at once. For very long videos, ask for a specific section rather than summarising the whole thing.

Tip 3: Use with Obsidian

Save processed summaries to your notes for future reference. This builds a searchable knowledge base of video content you've consumed.

Tip 4: English-only is the default

The skill works best with English-language videos. Non-English auto-captions vary in quality.

Key Lessons Learned

  • yt-dlp is remarkably robust. It handles shortened URLs (youtu.be/), playlist entries, and timestamped links without issue.
  • There's no bulk processing built in, but you can write a simple loop to process multiple videos.
  • The skill runs synchronously — the agent waits for the transcript before responding. For long transcripts this takes a few seconds.
  • No API key needed. yt-dlp works directly with YouTube's public endpoints.
  • The skill respects YouTube's rate limiting. Don't hammer it with hundreds of requests in quick succession.
